2. Cart Rules
    2.1. Users can add items to the cart using the "Add to Cart" button.
    2.2. If the "Add to Cart" button is clicked again, the quantity of the item increases.
    2.3. The quantity of items can also be increased by manually updating the quantity in the input box specified for quantity.
    2.4. If an item needs to be removed, the user can click the "Remove" button.
    2.5. The cart displays the "Item name", "Unit price", "Quantity of the item".
    2.6. If there are no items in the cart, the cart displays "Cart is empty".

3. Pricing Rules
    3.1. The price is displayed after the cart items are displayed.
    3.2. The total price is calculated as (sum of ("price for one item" x "quantity of item")).
    3.3. The total price is subject to change with respect to the dicount code and shipping charges if applied.

4. Discount Code Rules
    4.1. The valid Discount Codes are: "DISCOUNTFOR16" & "HALFPRICEOFF"
    4.2. The discount code "DISCOUNTFOR16" applies a 16% discount.
    4.3. The discount code "HALFPRICEOFF" applied a 50% discount.
    4.4. Only one discount code can be applied at a time.
    4.5. Once the discount code is applied, the changes will be reflected in the total price.

5. Shipping Rules
    5.1. The available shipping is "Standard" and "Express"
    5.2. The "Standard" shipping takes a delivery time of 5-7 days and is free of cost.
    5.3. The "Express" shipping takes a delivery time of 1-2 days and costs an extra Rs. 100.
    5.4. The default mode of shipping is "Standard".
    5.5. If the "Express" shipping mode is selected, the changes will be reflected on the total price.

6. User Details Requirements
    6.1. Users must specify their details in the "Name", "Email", "Address" fields.
    6.2. The "Email" field must follow the pattern: "[letter][letters|numbers|dot|underscore|percent|plus|hyphen]@[letters|numbers|dot|hyphen].[letters(atleast 2-letter extension)]".
    6.3. If the "Name" field is left empty, "Name is required" will be displayed implying that the user must enter the appropriate value.
    6.4. If the "Email" field is left empty or if there is a mismatch in the email format, "Invalid email" will be displayed implying that the user must enter the appropriate value.
    6.5. If the "Address" field is left empty, "Address is required" will be displayed implying that the user must enter the appropriate value.

7. Payment Rules
    7.1. The supported methods of payment are "Credit Card" and "PayPal".
    7.2. If the form is valid and the cart is not empty, the text "Payment Successful!" is displayed.
    7.3. If the cart is empty, the alert box relays the message "Your cart is empty!".
    7.4. If the form fields do not satisfy the conditions, the respective inline messages are displayed and the payment is not processed.
